Training API

The Training API provides utilities for training and fine-tuning trajectory prediction models.

Training Configuration

build_and_train_model(X_seq_train, X_lidar_train, y_train, seq_length, n_features, n_lidar_features, prediction_horizon, validation_split=0.1, batch_size=32, epochs=50)

Build and train a trajectory prediction model with best practices.

Parameters:

  • X_seq_train (numpy.ndarray) – Training sequence data

  • X_lidar_train (numpy.ndarray) – Training LiDAR data

  • y_train (numpy.ndarray) – Training target data

  • seq_length (int) – Length of input sequence

  • n_features (int) – Number of features in sequence data

  • n_lidar_features (int) – Number of features in LiDAR data

  • prediction_horizon (int) – Number of future steps to predict

  • validation_split (float) – Fraction of data to use for validation

  • batch_size (int) – Training batch size

  • epochs (int) – Maximum number of training epochs

Returns:

Trained model and training history

Return type:

tuple

# Example usage
model, history = build_and_train_model(
    X_seq_train, X_lidar_train, y_train,
    seq_length=10, n_features=15, n_lidar_features=4, prediction_horizon=10
)
initialize_model_for_training(model, learning_rate=0.001)

Initialize model for training with appropriate optimizer and loss.

Parameters:

  • model (tensorflow.keras.Model) – Model to initialize

  • learning_rate (float) – Initial learning rate

Returns:

Compiled model

Return type:

tensorflow.keras.Model

Training Callbacks

The framework provides several training callbacks for optimal training:

Early Stopping

Prevents overfitting by monitoring validation loss and stopping when it stops improving.

early_stopping = keras.callbacks.EarlyStopping(
    monitor='val_loss',
    patience=10,
    restore_best_weights=True,
    verbose=1
)

Learning Rate Scheduling

Adapts the learning rate during training to improve convergence.

lr_scheduler = keras.callbacks.ReduceLROnPlateau(
    monitor='val_loss',
    factor=0.5,
    patience=5,
    min_lr=1e-6,
    verbose=1
)

Model Checkpointing

Saves the best model during training based on validation performance.

checkpoint = keras.callbacks.ModelCheckpoint(
    'trajectory_model_best.h5',
    monitor='val_loss',
    save_best_only=True,
    verbose=1
)

Hyperparameter Tuning

tune_learning_rate(X_seq_train, X_lidar_train, y_train, X_seq_val, X_lidar_val, y_val, model, min_lr=1e-5, max_lr=1e-1, steps=20)

Perform learning rate range test to find optimal learning rate.

Parameters:

  • X_seq_train (numpy.ndarray) – Training sequence data

  • X_lidar_train (numpy.ndarray) – Training LiDAR data

  • y_train (numpy.ndarray) – Training target data

  • X_seq_val (numpy.ndarray) – Validation sequence data

  • X_lidar_val (numpy.ndarray) – Validation LiDAR data

  • y_val (numpy.ndarray) – Validation target data

  • model (tensorflow.keras.Model) – Model to tune

  • min_lr (float) – Minimum learning rate to test

  • max_lr (float) – Maximum learning rate to test

  • steps (int) – Number of learning rates to test

Returns:

Optimal learning rate

Return type:

float

Data Augmentation

augment_sequence_data(X_seq, y_seq, jitter_factor=0.05, dropout_prob=0.1)

Augment sequence data with noise and feature dropout for improved robustness.

Parameters:

  • X_seq (numpy.ndarray) – Input sequence data

  • y_seq (numpy.ndarray) – Target sequence data

Returns:

Augmented input and target data

Return type:

tuple

Augmentation techniques:

  • Random jittering: Adding noise to continuous features

  • Feature dropout: Randomly setting features to zero

  • Sequence shifting: Shifting sequence by small offsets

  • Target jittering: Adding controlled noise to target values

augment_lidar_data(X_lidar, rotation_range=15, translation_range=0.2)

Augment LiDAR point cloud data with geometric transformations.

Parameters:

X_lidar (numpy.ndarray) – Input LiDAR data

Returns:

Augmented LiDAR data

Return type:

numpy.ndarray

Augmentation techniques:

  • Random rotation: Rotating point cloud around z-axis

  • Random translation: Shifting point cloud by small offsets

  • Point dropout: Randomly removing points

  • Intensity jittering: Adding noise to intensity values

Cross-Validation

perform_cross_validation(X_seq, X_lidar, y_seq, seg_info, model_fn, n_folds=5)

Perform k-fold cross-validation for robust model evaluation.

Parameters:

  • X_seq (numpy.ndarray) – Sequence data

  • X_lidar (numpy.ndarray) – LiDAR data

  • y_seq (numpy.ndarray) – Target data

  • seg_info (list) – Segment information

  • model_fn (callable) – Function to create model

  • n_folds (int) – Number of folds

Returns:

Cross-validation results

Return type:

dict

This function ensures that data from the same driving segment is not split between training and validation sets.

Training Visualization

plot_training_history(history)

Plot training and validation metrics from model training history.

Parameters:

history (tensorflow.keras.callbacks.History) – Training history

Plots include:

  • Training vs. validation loss

  • Learning rate schedule

  • Metrics breakdown by epoch

# Example usage
plt.figure(figsize=(10, 5))
plt.plot(history.history['loss'], label='Training Loss')
plt.plot(history.history['val_loss'], label='Validation Loss')
plt.title('Model Training History')
plt.xlabel('Epoch')
plt.ylabel('Loss')
plt.legend()
plt.grid(True)
plt.savefig('training_history.png')
plt.show()
