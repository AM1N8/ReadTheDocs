Data API

The Data API provides utilities for loading, processing, and managing trajectory data from the CARLA simulator.

Data Loading

load_lidar_point_cloud(file_path)

Load LiDAR point cloud from a PLY file.

Parameters:

file_path (str) – Path to the PLY file

Returns:

Numpy array of shape (N, 4) containing x, y, z coordinates and intensity

Return type:

numpy.ndarray

# Example usage
point_cloud = load_lidar_point_cloud("lidar_1234567890.ply")
print(f"Loaded {len(point_cloud)} points with shape {point_cloud.shape}")
map_timestamps_to_lidar(ego_df, lidar_path)

Map timestamps from ego vehicle data to corresponding LiDAR files.

Parameters:

  • ego_df (pandas.DataFrame) – DataFrame containing ego vehicle data with ‘timestamp’ column

  • lidar_path (str) – Path to directory containing LiDAR files

Returns:

Dictionary mapping timestamps to LiDAR file paths

Return type:

dict

load_camera_image(file_path)

Load camera image from file (PNG, JPEG).

Parameters:

file_path (str) – Path to the image file

Returns:

RGB image as numpy array of shape (H, W, 3)

Return type:

numpy.ndarray

# Example usage
image = load_camera_image("rgb_1234567890.png")
print(f"Loaded image with shape {image.shape}")
map_timestamps_to_images(ego_df, image_path, camera_type='rgb')

Map timestamps from ego vehicle data to corresponding camera images.

Parameters:

  • ego_df (pandas.DataFrame) – DataFrame containing ego vehicle data with ‘timestamp’ column

  • image_path (str) – Path to directory containing image files

  • camera_type (str) – Type of camera images (‘rgb’, ‘depth’, ‘semantic’)

Returns:

Dictionary mapping timestamps to image file paths

Return type:

dict

Point Cloud Processing

sample_point_cloud(points, n_points=1024)

Sample a fixed number of points from a point cloud with improved sampling strategy.

Parameters:

  • points (numpy.ndarray) – Input point cloud array of shape (N, D)

  • n_points (int) – Number of points to sample (default: 1024)

Returns:

Sampled point cloud of shape (n_points, D)

Return type:

numpy.ndarray

Key features:

  • Distance-based weighting prioritizes points closer to the vehicle

  • Handles point clouds with fewer points than requested

  • Ensures consistent output dimension regardless of input size

process_point_cloud(points, max_points=1024)

Process point cloud data with advanced normalization techniques.

Parameters:

  • points (numpy.ndarray) – Input point cloud array

  • max_points (int) – Maximum number of points (default: 1024)

Returns:

Processed point cloud with normalized features

Return type:

numpy.ndarray

Processing steps:

  1. Center points using robust median centering

  2. Scale to unit sphere using 90th percentile normalization

  3. Normalize intensity values to [0,1] range

Vehicle Data Processing

calculate_vehicle_dynamics(ego_df)

Calculate enhanced vehicle dynamics features from raw ego vehicle data.

Parameters:

ego_df (pandas.DataFrame) – DataFrame with basic vehicle data

Returns:

DataFrame with additional dynamics features

Return type:

pandas.DataFrame

Computed features include:

  • Speed (magnitude of velocity)

  • Heading (direction of travel)

  • Acceleration components and magnitude

  • Jerk (rate of change of acceleration)

  • Heading change rate

  • Curvature

  • Movement state (binary indicator)

calculate_relative_displacement(ego_df)

Calculate relative displacements between consecutive frames.

Parameters:

ego_df (pandas.DataFrame) – DataFrame with vehicle positions

Returns:

DataFrame with added relative displacement columns

Return type:

pandas.DataFrame

filter_and_segment_data(ego_df, min_speed=0.5, min_segment_length=5)

Filter stationary data and segment into meaningful driving sequences.

Parameters:

  • ego_df (pandas.DataFrame) – DataFrame with vehicle dynamics

  • min_speed (float) – Minimum speed threshold for “moving” state

  • min_segment_length (int) – Minimum length of a valid segment

Returns:

Filtered DataFrame with segment information

Return type:

pandas.DataFrame

Image Processing

process_camera_image(image, target_size=(224, 224), normalize=True)

Process camera image for model input with resizing and normalization.

Parameters:

  • image (numpy.ndarray) – Input RGB image

  • target_size (tuple) – Target image dimensions (height, width)

  • normalize (bool) – Whether to normalize pixel values to [0,1]

Returns:

Processed image

Return type:

numpy.ndarray

Processing steps:

  1. Resize to target dimensions

  2. Convert to float32 data type

  3. Normalize pixel values if requested

  4. Handle different input formats (RGB, grayscale, etc.)

extract_road_features(semantic_image)

Extract road features from semantic segmentation image.

Parameters:

semantic_image (numpy.ndarray) – Semantic segmentation image

Returns:

Binary road mask and road boundary features

Return type:

tuple

augment_camera_images(images, augmentation_strength=0.5)

Apply data augmentation techniques to camera images.

Parameters:

  • images (numpy.ndarray) – Array of images

  • augmentation_strength (float) – Strength of augmentation (0.0 to 1.0)

Returns:

Augmented images

Return type:

numpy.ndarray

Augmentation techniques:

  • Random brightness and contrast adjustment

  • Random color jitter

  • Random blur and noise addition

  • Random horizontal flip (for training only)

plt.title("Town05 Map")

Sequence Preparation

prepare_relative_sequences(df_processed, timestamp_to_lidar, seq_length=10, prediction_horizon=5, min_speed=0.5)

Prepare input sequences with relative displacement representation for model training.

Parameters:

  • df_processed (pandas.DataFrame) – Processed vehicle data

  • timestamp_to_lidar (dict) – Mapping from timestamps to LiDAR files

  • seq_length (int) – Length of input sequence

  • prediction_horizon (int) – Number of future steps to predict

  • min_speed (float) – Minimum average speed threshold

Returns:

tuple containing (input sequences, LiDAR data, target sequences, input scaler, target scaler, segment info)

Return type:

tuple

The returned data is structured as:

  • X_seq: Array of shape (N, seq_length, n_features)

  • X_lidar: Array of shape (N, 1024, 4)

  • y_seq: Array of shape (N, prediction_horizon, 2)

  • scaler_input: StandardScaler for input normalization

  • scaler_target: StandardScaler for target normalization

  • segment_info: List of dictionaries with segment metadata

prepare_multimodal_sequences(df_processed, timestamp_to_lidar, timestamp_to_image, map_data, seq_length=10, prediction_horizon=5)

Prepare multimodal input sequences combining vehicle dynamics, LiDAR, and camera data.

Parameters:

  • df_processed (pandas.DataFrame) – Processed vehicle data

  • timestamp_to_lidar (dict) – Mapping from timestamps to LiDAR files

  • timestamp_to_image (dict) – Mapping from timestamps to camera image files

  • map_data (object) – Map data object

  • seq_length (int) – Length of input sequence

  • prediction_horizon (int) – Number of future steps to predict

Returns:

tuple containing multimodal data components

Return type:

tuple

The returned data includes:

  • Vehicle dynamics sequences

  • LiDAR point cloud sequences

  • Camera image sequences

  • Map image patches centered on the vehicle

  • Target trajectories

Data Collection

setup_carla_client(host='localhost', port=2000, timeout=10.0)

Setup connection to CARLA simulator.

Parameters:

  • host (str) – Host address of CARLA server

  • port (int) – Port number

  • timeout (float) – Connection timeout in seconds

Returns:

CARLA client instance

Return type:

carla.Client

configure_sensors(vehicle, output_path, lidar_freq=10, camera_freq=10)

Configure and attach sensors to a vehicle for data collection.

Parameters:

  • vehicle (carla.Vehicle) – Vehicle actor in simulation

  • output_path (str) – Path to save sensor data

  • lidar_freq (int) – LiDAR capture frequency in Hz

  • camera_freq (int) – Camera capture frequency in Hz

Returns:

Dictionary of attached sensors

Return type:

dict

Configured sensors include:

  • RGB camera (front-facing)

  • Depth camera (front-facing)

  • Semantic segmentation camera (front-facing)

  • LiDAR sensor (roof-mounted)

  • GNSS sensor for positioning

collect_vehicle_data(client, map_name, duration=300, num_vehicles=50, save_path='./data')

Collect comprehensive vehicle trajectory and sensor data in CARLA.

Parameters:

  • client (carla.Client) – CARLA client instance

  • map_name (str) – Name of the map to use

  • duration (int) – Data collection duration in seconds

  • num_vehicles (int) – Number of background vehicles

  • save_path (str) – Path to save collected data

Returns:

Path to saved data

Return type:

str

Data collection process:

  1. Load specified map and set weather conditions

  2. Spawn autopilot ego vehicle with sensors

  3. Spawn background traffic

  4. Record vehicle state, sensor data, and ground truth trajectories

  5. Process and organize collected data
